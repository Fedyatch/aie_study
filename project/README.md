# Итоговый проект по курсу «Инженерия Искусственного Интеллекта»

В этой папке находится итоговый мини-проект по курсу.  
Проект должен демонстрировать применение методов и инструментов инженерии ИИ: работу с данными, модели, пайплайны, сервис, эксперименты и (по возможности) воспроизводимость.

## 1. Паспорт проекта

- **Название проекта:** Сервис кредитного скоринга: предсказание вероятности дефолта заёмщика
- **Автор:** Чаусов Фёдор Валерьевич
- **Группа:** КМБО-11-24
- **Контакт:** fedyatchausov@yandex.ru

- **Краткое описание (2-4 предложения):**  
  End-to-end сервис оценки кредитного риска на основе табличных данных. Задача бинарной классификации решается с помощью ансамблевых моделей (Logistic Regression, Random Forest, HistGradientBoosting), обученных на German Credit Data (Statlog). Реализован REST API на FastAPI, который по анкетным данным возвращает вероятность дефолта, категорию риска и время обработки. Архитектура включает модульный пайплайн предобработки, контейнеризацию через Docker, автоматические тесты и документированный процесс экспериментов.

## 4. Как запустить проект

### 4.1. Запуск обучения модели

Обучение проводится в Jupyter-ноутбуках:


```bash
cd project
uv run jupyter lab
```

Затем открыть и выполнить последовательно:
- `notebooks/01_eda_baseline.ipynb` — EDA и baseline
- `notebooks/02_model_tuning.ipynb` — обучение моделей, сохранение артефактов

### 4.2. Запуск сервиса (API/веб-интерфейс)

```bash
# Локальный запуск с горячей перезагрузкой
cd project
uv run uvicorn src.credit_scoring.api:app --reload
```

**Сервис доступен:**
- Основной адрес: `http://127.0.0.1:8000`
- Swagger UI (документация): `http://127.0.0.1:8000/docs`
- Health-check: `http://127.0.0.1:8000/health`

### 4.3. Запуск в Docker

```bash
# Сборка образа
cd project
docker build -t credit-scoring-api .

# Запуск контейнера
docker run -p 8000:8000 credit-scoring-api
```

## 5. Данные

- **Источник:** Открытый датасет [German Credit Data (Statlog)](https://archive.ics.uci.edu/ml/datasets/statlog+(german+credit+data)), UCI Repository.
- **Объём:** 1000 записей, 20 признаков (13 категориальных, 7 числовых).

- **Дисбаланс классов:** ~70% хороших, ~30% плохих.
- **Расположение:** `data/german_credit.csv` (небольшой файл, хранится в репозитории).

> Для обучения используется открытый датасет German Credit Data.

## 7. Модели и эксперименты

### 7.1. Сравнение моделей

| Модель | ROC-AUC | Accuracy | F1 | Время обучения |
|--------|---------|----------|----|---------------|
| **Logistic Regression** (финальная) | **0.78** | **0.75** | **0.54** | 0.2 с |
| Random Forest | 0.80 | 0.76 | 0.56 | 1.5 с |
| HistGradientBoosting | 0.80 | 0.76 | 0.56 | 0.8 с |

### 7.2. Обоснование выбора Logistic Regression

1. **Качество**: ROC-AUC = 0.78 при небольшом датасете (1000 записей) — достаточный уровень для демонстрации полного цикла.
2. **Производительность**: инференс < 10 мс — идеально для real-time API.
3. **Интерпретируемость**: коэффициенты модели позволяют объяснять решения, что критично в финансовой сфере.
4. **Простота и устойчивость**: низкий риск переобучения на малых данных, отсутствие необходимости в сложной настройке гиперпараметров.

Подробное обоснование — в `report.md`, раздел «Выбор финальной модели».

## 9. Ограничения и дальнейшая работа

### 9.1. Текущие ограничения

| Ограничение | Влияние | Обходное решение |
|-------------|---------|------------------|
| Малый объём данных (1000 записей) | Снижение обобщающей способности | Сбор дополнительных данных, использование более крупных датасетов |
| Отсутствие калибровки вероятностей | Вероятности не отражают реальную частоту дефолтов | Применение `CalibratedClassifierCV` |
| Отсутствие мониторинга дрейфа данных | Риск деградации качества со временем | Интеграция Evidently AI или Prometheus-алертов |
| Базовая валидация (hold-out) | Возможна нестабильность оценки | Кросс-валидация на временных рядах или стратифицированная KFold |
| Отсутствие аутентификации в API | Сервис открыт для всех | Добавление JWT-токенов или API-ключей |

### 9.2. Направления развития

1. **Интерпретируемость**: Интеграция SHAP для локальной интерпретации предсказаний (`/explain` endpoint).
2. **Мониторинг**: Добавление Prometheus-метрик (`/metrics`) и алертов при деградации качества.
3. **MLOps**: Автоматический пайплайн переобучения через MLflow/Airflow и A/B-тестирование версий моделей.
4. **Масштабирование**: Горизонтальное масштабирование через Kubernetes с readiness/liveness probes.
5. **Безопасность**: Валидация входных данных на уровне WAF, rate limiting, аудит логов.
