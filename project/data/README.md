# Данные проекта

В этой директории находятся данные, необходимые для обучения модели и работы сервиса кредитного скоринга.

## 1. Правила хранения данных

Согласно требованиям курса и политике приватности:

**Допустимо хранить:**
- Инструкции по загрузке открытых датасетов.
- Небольшие примеры (sample) для проверки структуры.
- Обезличенные учебные выборки.

**Запрещено хранить в репозитории:**
- Реальные персональные или конфиденциальные данные.
- Полные «сырые» файлы датасетов объёмом более 50 МБ.
- Файлы с секретами, токенами или учётными данными.

Исходные CSV-файлы исключены из контроля версий через `.gitignore`.

---

## 2. Структура директории
```text
data/
├── README.md              # Этот файл: описание данных и инструкция по загрузке
└── raw/                   # Исходные файлы датасета (игнорируются в .gitignore)
    └── cs-training.csv    # Основной тренировочный набор (~150 000 записей)
```

Опционально можно также скачать `cs-test.csv`, `Data Dictionary.xls` и `sampleEntry.csv` с Kaggle, но для проекта они не требуются — все эксперименты и разбиение на train/test выполняются внутри ноутбука на основе `cs-training.csv`.

## 3. Описание датасета

| Параметр | Значение |
|----------|----------|
| **Название** | Give Me Some Credit |
| **Источник** | [Kaggle Competition](https://www.kaggle.com/c/GiveMeSomeCredit) |
| **Объём** | ~150 000 записей |
| **Признаки** | 10 числовых (финансовые показатели, история просрочек, демография) |
| **Целевая переменная** | `SeriousDlqin2yrs` ∈ {0, 1} (серьёзная просрочка 90+ дней в течение 2 лет) |
| **Тип данных** | Открытый, полностью обезличенный, учебный |
| **Лицензия** | Разрешено использование в некоммерческих и образовательных целях |

Датасет не содержит персональных идентификаторов (ФИО, паспортные данные, контакты). Все значения представлены в агрегированном или нормализованном виде.

## 4. Инструкция по загрузке

Для локального запуска проекта необходимо самостоятельно скачать тренировочный файл:

1. Перейдите на страницу соревнования: https://www.kaggle.com/c/GiveMeSomeCredit
2. Примите правила соревнования (кнопка `Rules` → `Accept`).
3. В разделе `Data` скачайте `cs-training.csv`.
4. Поместите файл в директорию `data/raw/`.
5. Убедитесь, что файл не попадает под версионный контроль (правило в `.gitignore` уже настроено).

После этого все ноутбуки (`01_eda_baseline.ipynb`, `02_model_tuning.ipynb`) смогут загрузить данные и корректно отработать.

---

## 5. Использование в проекте

Данные обрабатываются в два этапа:

| Этап | Где происходит | Описание |
|------|----------------|----------|
| **Загрузка и EDA** | `notebooks/01_eda_baseline.ipynb` | Чтение CSV, анализ пропусков, распределений, выбросов, baseline |
| **Предобработка и обучение** | `notebooks/02_model_tuning.ipynb` | Импутация, логарифмирование, агрегация просрочек, масштабирование, обучение моделей |
| **Инференс** | `src/credit_scoring/model.py` | Загрузка сохранённого пайплайна (`scorer_pipeline_v1.0.joblib`), который уже содержит все шаги предобработки |

Таким образом, после обучения модели в ноутбуке сервис **не зависит от сырых данных** — он использует только сериализованный пайплайн.

---

## 6. Ограничения и воспроизводимость

- **Объём**: Полный датасет занимает ~12 МБ в сжатом виде и ~45 МБ в разархивированном. Хранение в Git нецелесообразно.
- **Воспроизводимость**: Инструкции в `README.md` и `project/README.md` позволяют восстановить рабочую среду за несколько минут.
- **Изменения**: Структура признаков зафиксирована в коде предобработки и в сохранённом пайплайне. При обновлении версии датасета на Kaggle может потребоваться переобучение модели.

Для вопросов по данным или структуре пайплайна обращайтесь к документации в `project/report.md` или к модулям в `src/credit_scoring/`.
