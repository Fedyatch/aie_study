# Конфигурационные файлы

В этой папке хранятся **настройки проекта**:

- Шаблоны переменных окружения:
  - `.env.example` — список переменных **без** реальных значений (секретов, токенов, паролей).
- (Опционально) файлы с параметрами:
  - `config.yaml` — общие настройки проекта;
  - `training.yaml` — параметры обучения;
  - `inference.yaml` — параметры инференса.

## Важные правила

- **Никогда** не храните в репозитории реальные файлы `.env` с паролями, токенами или ключами. Они должны быть только локально и добавлены в `.gitignore`.
- **Всегда** предоставляйте шаблон `.env.example`, чтобы пользователь мог скопировать его в `.env` и заполнить своими значениями.
- Конфигурационные файлы (`.yaml`, `.json`) могут содержать пути к данным, гиперпараметры модели, параметры обучения и инференса.

## Текущее содержимое

- `.env.example` — шаблон с переменной `PORT=8000` (порт, на котором запускается сервис).

## Как использовать

1. Скопируйте шаблон в реальный конфигурационный файл:
   ```bash
   cp .env.example .env
